# HyperMem 0.8.5 Release Post

## GitHub Post

HyperMem 0.8.5 is live.

This release does 2 important things.

First, it makes failover detection provider/model-aware. HyperCompositor now distinguishes provider swaps, model swaps, and real budget changes, so model-state drift is detected correctly without creating extra churn.

Second, it cleans up the install story. README, INSTALL, and TUNING now describe the npm-first path more clearly, separate staging from activation, tighten `plugins.allow` guidance, and make first-run verification easier for both operators and agents.

What shipped:
- provider/model-aware failover detection in HyperCompositor
- persisted model state now includes `modelKey`, `provider`, and `modelId`
- verbose logging now surfaces provider/model changes even when budget does not downshift
- declarative install docs for npm-first OpenClaw setup
- clearer verification states for staged, wired, loaded, and healthy-but-empty installs
- release parity across the core package and both plugins

npm packages now live at 0.8.5:
- `@psiclawops/hypermem`
- `@psiclawops/hypercompositor`
- `@psiclawops/hypermem-memory`

Install:

```bash
npm install @psiclawops/hypermem@0.8.5
npx hypermem-install
```

Source, tuning, and full setup docs:
- https://github.com/PsiClawOps/hypermem
- https://github.com/PsiClawOps/hypermem/blob/main/INSTALL.md

## Discord Post

HyperMem 0.8.5 is out.

This one is mostly about making the product behave more honestly under real operator conditions.

On the runtime side, HyperCompositor now tracks provider and model identity explicitly. That means failover detection can tell the difference between a provider swap, a model swap, and a real budget change. You get better model-state detection without unnecessary downshift churn.

On the install side, we tightened the docs around the path real users actually take. README, INSTALL, and TUNING now separate package staging from actual activation, make the OpenClaw wiring steps more declarative, clarify merge-safe `plugins.allow` setup, and make it much easier to tell whether you are staged, wired, healthy, or just empty.
