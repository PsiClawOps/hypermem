# HyperMem 0.9.5 Release Notes

## Install

**npm:**
```bash
npm install @psiclawops/hypermem@0.9.5
```

**From source:**
```bash
git clone https://github.com/PsiClawOps/hypermem.git
cd hypermem && npm install && npm run build
```

See [INSTALL.md](../../INSTALL.md) for full setup instructions.

---

## Changes

- Fresh installs default to `embedding.provider: "none"`, so baseline installation does not require Ollama or a hosted embedding key.
- README, INSTALL, and TUNING now define install readiness as GREEN/YELLOW/RED. FTS5-only installs are YELLOW: usable baseline recall, semantic vector recall not yet enabled.
- Semantic recall setup is documented as an opt-in follow-up after the baseline install is verified.
- OpenClaw examples now use `openclaw gateway` commands.
- npm package payload now excludes internal architecture and release-process docs from the public artifact.
- Fresh-install smoke validation covers the FTS5/no-embedding default path.
